## GreenPrj 用户使用指南（Web 版本）

### 1. 安装与运行

- **前提**：已安装 Node.js 16+、npm 8+，以及现代浏览器（推荐 Chrome / Edge）。
- **步骤**：
  1. 克隆仓库：`git clone https://github.com/kevinchen202008-cmyk/Android-and-IOS-program-GreenPrj.git`
  2. 进入 Web 目录：`cd Android-and-IOS-program-GreenPrj/web`
  3. 安装依赖：`npm install`
  4. 启动开发服务器：`npm run dev`，浏览器访问 `http://localhost:5173`（或终端提示的地址）。

### 2. 首次使用流程（本地单机记账）

1. **设置访问密码（可选但推荐）**  
   - 首次进入应用时，按照提示设置密码。  
   - 密码用于本地数据保护，不会上传到云端。

2. **登录与解锁**  
   - 之后再次进入应用时，输入密码解锁本地账本。  
   - 默认会话时长为 **30 分钟**，超时后需要重新登录。

3. **创建第一笔账目**  
   - 在「记账」页面点击 **“新增账目”**。  
   - 输入金额、日期、类别（如餐饮、交通等）、备注后保存。  
   - 列表中可以查看、编辑、删除账目。

4. **统计与预算查看**  
   - 「统计」页面：查看按时间（天/周/月/年）和类别的消费统计、趋势图、类别分布。  
   - 「预算」页面：设置月度 / 年度预算，并查看预算执行情况。

### 3. 数据安全与备份 / 恢复

- **本地优先存储**：  
  - Web 版本默认使用浏览器本地的 IndexedDB 存储账目和设置数据；  
  - 数据不会自动上传云端，完全保存在当前设备浏览器中。

- **数据导出（备份）**：  
  - 前往「数据管理」页面，使用 **导出 JSON / CSV** 功能，将账本导出到本地文件；  
  - 建议定期备份，并妥善保存导出文件。

- **数据导入（恢复）**：  
  - 在「数据管理」页面选择 **导入备份文件**，选择之前导出的 JSON 文件；  
  - 导入前会进行数据校验与冲突检查，确认后再写入本地数据库。

- **数据删除**：  
  - 如需清空数据，可在「数据管理」中使用 **清除所有数据** 功能；  
  - 删除操作不可恢复，请务必确认已完成备份。

### 4. 常见问题（FAQ）

- **Q1：更换浏览器或设备后，数据会自动同步吗？**  
  A：当前阶段 Web 版本是 **本地单机版**，数据不会自动同步。  
  建议在旧设备上导出备份文件，再在新设备导入。

- **Q2：忘记密码怎么办？**  
  A：出于安全考虑，目前不支持找回原密码。  
  如忘记密码，只能通过 **清除本地数据** 重新开始（请提前导出重要数据）。

- **Q3：数据会上传到云端吗？**  
  A：不会。当前版本所有账本数据只保存在本地浏览器。  
  未来若引入云后端，会在文档和 UI 中明确提示并提供选择。

- **Q4：浏览器卸载 / 清理缓存会怎样？**  
  A：若清空浏览器存储或卸载浏览器，本地数据可能会被删除。  
  建议在重要操作前先 **导出备份**。
